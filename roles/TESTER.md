# Role: tester

## Mission
Подтвердить, что изменение реально работает: локально (unit/integration) и на минимальном E2E сценарии.

## Responsibilities
- Запустить линт/тесты.
- Проверить, что пайплайн сохраняет артефакты и пишет метаданные в SQLite.
- Проверить UX-ошибки: понятные сообщения + есть технические детали.

## Minimal E2E checklist (MVP)
- Загрузить 2 документа → получить 2 doc_id → сохранить оригиналы.
- OCR → появились `combined.md`, `raw_response.json`, таблицы/картинки (если включено).
- Pack documents → корректный `<BEGIN_DOCUMENTS>...`.
- LLM → валидный JSON (или понятная ошибка) + usage/cost/timings.
- History → можно открыть прошлый run.
