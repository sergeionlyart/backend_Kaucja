# Преамбула проекта

## Что это за этап

Проект переходит от подготовки правового корпуса к следующему контуру: анализ пользовательского кейса по спору о возврате арендной кауции и подготовка стратегического меморандума с прозрачной ссылочной системой.

На этом этапе источниками анализа являются:
- сообщение пользователя;
- пакет его первичных документов в Markdown;
- локальная MongoDB-коллекция нормативных документов, ранее подготовленная `legal_docs_pipeline`.

## Что считается рабочим результатом

Система должна формировать:
- `CaseIssueSheet` по фактической стороне кейса;
- `ResearchBundle` по найденным правовым основаниям;
- `StrategicMemo` с индексами правовых и доказательственных якорей;
- `MemoQcReport` для проверки целостности ссылок.

В текущем контуре не требуется:
- OCR;
- frontend-навигация по ссылкам;
- кликабельные ссылки;
- промышленная sentence-level citation runtime.

## Архитектурный принцип

Решение строится как контролируемый pipeline на OpenAI Agents SDK:
1. `CaseIntakeAgent`
2. `LegalResearchAgent`
3. `MemoWriterAgent`
4. `CitationQCAgent` — как отдельный QC-шаг, а не как обязательный handoff-механизм

Нормативная база доступна агенту только через ограниченный tooling-слой для MongoDB. Агент не имеет прямого доступа к базе и не выполняет произвольные запросы.

## Источник истины

На старте проекта источником истины по требованиям является папка:

`docs/Tech_Spec_Agents_Legal`

Работа должна вестись на основании документации из этой папки с учётом дополнений из текущего supplement-пакета.

## Почему нужен supplement

Исходный комплект документации даёт хорошую архитектурную основу, но до старта разработки в нём оставались пробелы:
- отсутствовал отдельный prompt для `LegalResearchAgent`;
- не были формализованы Pydantic-модели для `CaseIssueSheet`, `ResearchBundle`, `StrategicMemo`, `MemoQcReport`;
- были несогласованности в именовании prompt-файлов и в роли верхнеуровневого агента;
- не был оформлен отдельный практический шаг по врезке документации в репозиторий.

Этот supplement закрывает именно эти стартовые пробелы.
