# Руководство пользователя (RU)

Это руководство для обычного пользователя: как загрузить документы, запустить анализ и понять результат.

## 1) Быстрый старт

1. Откройте приложение в браузере: `http://127.0.0.1:7400`.
2. В блоке **Documents** загрузите один или несколько файлов.
3. Выберите:
   - **Provider** (например, `openai` или `google`);
   - **Model**;
   - **Prompt Name / Prompt Version** (обычно оставляйте значения по умолчанию).
4. Нажмите кнопку **Analyze**.
5. Дождитесь завершения и смотрите результаты в блоках ниже.

## 2) Как пользоваться интерфейсом

### Загрузка документов

- Нажмите поле **Documents** и выберите файлы с компьютера.
- Можно загружать несколько файлов за один запуск.
- Рекомендуется загружать читаемые PDF/изображения; TXT также поддерживается.

### Запуск анализа

- После выбора провайдера/модели нажмите **Analyze**.
- Во время работы смотрите:
  - **Status** — общий статус;
  - **Progress (OCR -> LLM -> validate -> finalize)** — этап выполнения;
  - **Runtime Log Tail** — последние строки технического лога;
  - **User-friendly Error** и **Error Details** — если возникла ошибка.

### Где смотреть результаты

- **OCR Results** — список документов и статус OCR.
- **Checklist** — проверочный список фактов по делу.
- **Gap List** — что не хватает и что нужно запросить.
- **Details item_id / Details** — расшифровка конкретного пункта.
- **Summary** — краткий итог по критичным пробелам и следующим шагам.
- **Raw JSON** — полный структурированный ответ модели.
- **Validation** — результат проверки структуры ответа.
- **Metrics** — время, токены, стоимость.

### Дополнительные действия пользователя

- **Prompt Management**:
  - просмотр/редактирование `system_prompt.txt`;
  - сохранение новой версии через **Save as new version**.
- **Run History**:
  - фильтрация запусков (по session/provider/model/дате);
  - загрузка прошлого запуска через **Load Selected Run**.
- **Export run bundle (zip)**:
  - выгрузка артефактов выбранного запуска в ZIP.
- **Restore run bundle**:
  - восстановление запуска из ZIP;
  - режим **Verify only (no restore)** для проверки архива без записи.
- **Delete run**:
  - удаление запуска (опционально с резервной ZIP-копией).
- **Compare Selected Runs**:
  - сравнение двух запусков (изменения в checklist, gaps, метриках).

## 3) Как это работает (кратко, без кода)

1. Вы загружаете документы через UI.
2. Система создаёт новый запуск (`run`) и сохраняет служебные метаданные.
3. Документы проходят OCR (Mistral OCR):
   - PDF/изображения обрабатываются напрямую;
   - TXT предварительно конвертируется в PDF, затем идёт в OCR.
4. OCR-текст собирается и передаётся в языковую модель (OpenAI или Google) в строго структурированном формате.
5. Ответ модели проверяется по JSON-схеме и дополнительным правилам валидации.
6. Результаты и артефакты сохраняются (логи, JSON, OCR-файлы) и показываются в интерфейсе.

## 4) Типичные проблемы (FAQ)

### Ошибка `Runtime preflight failed`

Причина: обычно отсутствует API-ключ или пакет провайдера.

Что проверить:
- `MISTRAL_API_KEY` (обязателен для OCR),
- `OPENAI_API_KEY` и/или `GOOGLE_API_KEY` (для выбранного провайдера LLM),
- корректность `.env`.

### Нажал Analyze, но запуск не завершился

1. Проверьте **Status**, **User-friendly Error**, **Error Details**.
2. Откройте **Runtime Log Tail** и **Full Log Path**.
3. Убедитесь, что документ читаемый и не повреждён.
4. Повторите запуск с теми же файлами и другой моделью (при необходимости).

### Файл загрузился, но результат странный

- Проверьте **OCR Results** и **ocr_quality_warnings** в `Raw JSON`.
- Для плохих сканов/фото часть данных может быть помечена как `ambiguous`/`missing`.
- Используйте **Gap List** как список того, что нужно дозапросить.

### Не работает экспорт/восстановление ZIP

- Для экспорта нужен существующий `run_id` с артефактами.
- Для restore используйте целый архив run bundle.
- Если сомневаетесь, сначала включите **Verify only (no restore)**.

### Как понять, что всё успешно

Признаки корректного выполнения:
- `Status`: завершено без ошибок;
- в **Validation** нет ошибок;
- заполнены **Checklist**, **Summary**, **Metrics**;
- доступен `Run ID` и путь к артефактам.

## 5) Практический совет

Для ежедневной работы достаточно сценария:
**загрузили документы -> Analyze -> просмотрели Checklist/Gap List/Summary -> при необходимости экспортировали ZIP**.
