# Workflow: bugfix

## Шаги
1) Repro:
- минимальный сценарий воспроизведения,
- expected vs actual.

2) Add test first:
- добавьте тест, который падает.

3) Fix:
- минимальное изменение,
- не рефакторить «заодно».

4) Verify:
- тест зелёный,
- линт зелёный.

5) Postmortem (кратко):
- причина,
- как предотвратить повтор (доп. тест / инвариант / лог).
